# v0.2.0 Release Notes

**Release Date**: 2025-12-14

## Summary

This release formalizes image variants (`-slim` vs `-runner`), introduces a schema-driven baseline profile manifest for DRY policy enforcement, and adds manifest-driven license/notice validation.

## Changes

### Breaking: explicit image variants

- New canonical images:
  - `ghcr.io/fulmenhq/goneat-tools-slim`
  - `ghcr.io/fulmenhq/goneat-tools-runner`
  - `ghcr.io/fulmenhq/sbom-tools-slim`
  - `ghcr.io/fulmenhq/sbom-tools-runner`
- Compatibility aliases remain published:
  - `ghcr.io/fulmenhq/goneat-tools:*` aliases to `ghcr.io/fulmenhq/goneat-tools-runner:*`
  - `ghcr.io/fulmenhq/sbom-tools:*` aliases to `ghcr.io/fulmenhq/sbom-tools-runner:*`
- Legacy `v0.1.x` images remain available but are considered deprecated in favor of explicit `-slim`/`-runner` variants.

### Baseline profiles (DRY + enforceable)

- Added `manifests/profiles.json` to define baseline package sets (starting with `runner_baseline`).
- Added `scripts/validate-profiles.sh` / `make validate-profiles` to enforce:
  - `-runner` images include the runner baseline.
  - `-slim` images do not include runner-only baseline packages.

### Licenses and notices (manifest-driven)

- Extended `manifests/tools.json` to optionally declare curated license/notice expectations (`license_spdx`, `license_path`/`license_paths`, `notice_required`, `notice_path`).
- Added `scripts/validate-licenses.sh` / `make validate-licenses` to build images and assert curated `/licenses/...` and `/notices/...` paths exist.
- Enforced NOTICE where declared required (notably Trivy NOTICE and FulmenHQ goneat NOTICE).

### DX payload upgrades (goneat-tools)

- Added `goneat` v0.3.20 and `sfetch` v0.2.7 to the goneat-tools payload (both `-slim` and `-runner`).

## Verification

### Pull by tag (recommended explicit variant)

```bash
docker pull ghcr.io/fulmenhq/goneat-tools-slim:v0.2.0
docker pull ghcr.io/fulmenhq/goneat-tools-runner:v0.2.0
```

### Verify curated license paths (local)

```bash
make validate-licenses
```
