# v0.1.2 Release Notes

**Release Date**: 2025-12-08

## Highlights

- **New Image**: `sbom-tools` for SBOM generation and vulnerability scanning
- **Matrix Release Workflow**: Both images now build in parallel on tag push
- **Improved Makefile**: Better developer experience with `build-all`, `test-all`, `check-clean`

## Images

### sbom-tools (NEW)

SBOM generation and vulnerability scanning for the FulmenHQ ecosystem.

| Tool | Version | Purpose |
|------|---------|---------|
| syft | v1.18.1 | SBOM generation (CycloneDX, SPDX) |
| grype | v0.86.1 | Vulnerability scanning |

**Base**: `alpine:3.21` (multi-arch: amd64, arm64)
**Size**: ~148MB

```bash
# Generate SBOM
docker run --rm -v "$(pwd):/work" ghcr.io/fulmenhq/sbom-tools:v0.1.2 \
  -c "syft /work -o cyclonedx-json"

# Vulnerability scan
docker run --rm -v "$(pwd):/work" ghcr.io/fulmenhq/sbom-tools:v0.1.2 \
  -c "grype /work"
```

### goneat-tools

No changes to tool versions. Rebuilt with updated workflow.

## Release Workflow Changes

- Matrix strategy: `[goneat-tools, sbom-tools]`
- Dockerfiles are SSOT for versions (no build-args in CI)
- Per-image artifacts uploaded to GitHub Release:
  - `sbom-goneat-tools-0.1.2.json`
  - `sbom-sbom-tools-0.1.2.json`
  - `SHA256SUMS-goneat-tools`
  - `SHA256SUMS-sbom-tools`

## Makefile Improvements

| Target | Description |
|--------|-------------|
| `build-all` | Build all images (single-arch) |
| `test-all` | Test all images |
| `check-clean` | Fail if working tree is dirty |
| `lint-dockerfiles` | Validate Dockerfile syntax (docker build --check) |
| `prepush` | Full validation: check-clean + quality + build-all + test-all |
